# 과제2 — Streamlit + GitHub 배포 완전 가이드

> 우리가 만든 **책 키워드 검색 에이전트(과제1)**를 Python(Streamlit) 웹앱으로 만들어
> GitHub에 올리고 Streamlit Cloud로 배포하는 전 과정입니다.
> 과제 방법 ①~④ 순서를 그대로 따릅니다.

---

## 준비물 (모두 무료)
- Google 계정 (Gemini API 키 발급용)
- GitHub 계정 — https://github.com 가입
- (선택) Cursor 또는 VS Code — 코드 확인·수정용

## ① 기획 및 명세 — "무엇을 만드는가"
과제 제출용 기능 명세는 이렇게 쓰면 됩니다(예시, 그대로 써도 됨):

> "경기도교육청 『부모, 학부모가 되다』 안내서를 학습한 AI 상담 도우미를 만든다.
> 좌측 사이드바에는 핵심 키워드 버튼·분야 필터·답변 길이 옵션을 두고,
> 중앙에는 챗봇 인터페이스를 배치한다. 학부모가 자연어로 질문하면
> 안내서 본문에서 관련 섹션을 검색해 Gemini API에 근거로 전달하고,
> 책 내용에 기반한 답변과 참고 페이지를 함께 보여준다."

→ 이 명세는 **이미 코드로 구현 완료**되어 있습니다(app.py).

## ② 바이브 코딩 구현 — (이미 완성된 코드 제공)
초안 생성·UI·반복개선 단계는 제공된 `app.py`로 충족됩니다.
Cursor/VS Code로 열어 내용을 확인하고, 원하면 AI 채팅으로
"답변 글씨를 키워줘" 같은 수정 요청을 해보면 '바이브 코딩' 경험이 됩니다.
(과제 취지상 직접 한두 번 수정해보는 걸 권장합니다.)

핵심 로직: 사용자 입력 → 책에서 관련 섹션 검색 → 그 내용을 프롬프트에 넣어
Gemini에 전달 → 책 기반 답변 출력. (프롬프트는 app.py의 build_prompt 함수)

## ③ 환경 설정 및 보안

### 3-1. Gemini API 키 발급
1. https://aistudio.google.com/app/apikey 접속 (구글 로그인).
2. **Create API key** 클릭 → 키 문자열 복사(AIza... 로 시작).
3. 이 키는 **절대 코드나 GitHub에 직접 넣지 마세요.** (아래 4-4에서 안전하게 입력)

### 3-2. requirements.txt (제공됨)
라이브러리 목록 파일. 이미 만들어져 있습니다:
```
streamlit
google-generativeai
```

### 3-3. 키 노출 방지 (제공됨)
- `.gitignore`에 secrets.toml이 등록되어 있어 키 파일은 업로드되지 않습니다.
- 실제 키는 secrets.toml이 아니라 **Streamlit Cloud 설정**에 넣습니다(4-4).

## ④ 배포 및 공유 — GitHub → Streamlit Cloud

### 4-1. GitHub 저장소 만들기
1. https://github.com 로그인 → 우측 상단 **+ > New repository**.
2. Repository name: 예) `parent-guide-ai`
3. **Public** 선택(Streamlit 무료 배포는 공개 저장소 권장).
4. **Create repository**.

### 4-2. 파일 업로드 (웹에서 가장 쉬움)
1. 만든 저장소 화면에서 **Add file > Upload files**.
2. 제가 드린 파일들을 끌어다 놓습니다:
   - `app.py`
   - `book_data.json`
   - `requirements.txt`
   - `README.md`
   - `.gitignore`
3. ⚠️ `.streamlit/secrets.toml.example`은 올려도 되지만, **실제 secrets.toml은 올리지 마세요.**
4. 아래 **Commit changes** 클릭.

> Cursor/VS Code의 Git 기능이나 `git push`로 올려도 됩니다. 웹 업로드가 가장 간단합니다.

### 4-3. Streamlit Cloud 연결
1. https://share.streamlit.io 접속 → **Continue with GitHub**로 로그인·권한 허용.
2. **Create app > Deploy a public app from GitHub**.
3. 설정:
   - Repository: 방금 만든 `본인계정/parent-guide-ai`
   - Branch: `main`
   - Main file path: `app.py`
4. **Deploy** 클릭. (1~3분 빌드)

### 4-4. API 키 안전하게 입력 (핵심 보안 단계)
1. 배포 화면 우측 하단 또는 앱 메뉴(⋮) > **Settings > Secrets**.
2. 입력창에 아래를 그대로 붙여넣고 키만 본인 것으로 교체:
   ```
   GEMINI_API_KEY = "AIza...본인키..."
   ```
3. **Save**. 앱이 자동 재시작되며 챗봇이 작동합니다.

### 4-5. 테스트 & 제출
1. 생성된 주소(`https://본인앱이름.streamlit.app`)로 접속.
2. **휴대폰·다른 PC**에서도 열어 정상 작동 확인(과제 요구사항).
3. 질문 예시로 테스트: "급식을 잘 안 먹어요", "늘봄학교가 뭔가요?", "학교폭력".
4. 최종 접속 링크를 **15주차**에 제출.

## 자주 막히는 곳
| 증상 | 해결 |
|---|---|
| 배포 후 "API 키가 설정되지 않았습니다" | 4-4의 Secrets 입력을 안 했거나 변수명이 다름. `GEMINI_API_KEY` 정확히 |
| 빌드 실패 (ModuleNotFound) | requirements.txt가 저장소 최상단에 있는지 확인 |
| book_data.json 못 찾음 | app.py와 같은 위치(최상단)에 함께 업로드했는지 확인 |
| 키를 실수로 GitHub에 올림 | aistudio에서 그 키 삭제 후 새 키 발급, Secrets에만 입력 |
